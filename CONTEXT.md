## 한 줄 정의
NetEase Cloud Music을 위한 **아티스트 발굴·평가·라이선스 인텔리전스 도구**.

## 회사 맥락
- **소속**: NetEase Cloud Music (网易云音乐)
- **하는 일**: A&R — 회사의 아티스트 풀 확장
- **최종 목표**: 더 좋은 아티스트(신인·기존 무관)를 발굴해 회사와 **라이선스 계약**을 맺게 한다.

## 우리가 푸는 문제
공개 스트리밍/소셜 + 외부 차트 데이터를 모아 아티스트를 분석하고, 일관된 기준으로 점수화해 **"라이선스할 만한 후보"를 추려내고 그 근거를 설명**한다. 사람이 일일이 손으로 뒤지는 발굴을 데이터로 거든다.

## "더 좋은 아티스트"의 기준 — A&R 논지 (4축)
점수는 단일 숫자가 아니다. 4개 축을 **각각 근거와 함께** 산출하고, **맥락 종합 코멘트**로 "왜 이 아티스트인가"의 서사를 만든다.

1. **성장세 / 모멘텀** — 최근 재생·팔로워·저장의 증가 속도. 이미 큰 스타보다 *"뜨는 중"*을 우선.
2. **참여 · 팬 충성도** — 재생 대비 좋아요·공유·댓글·재청취 등 *"관계의 깊이"*. 단순 규모가 아님.
3. **지역 / 장르 적합성** — NetEase 이용층(중국·아시아 시장)과의 장르·언어·정서 적합도. 플랫폼 매칭이 잘 될 아티스트.
4. **계약 가능성 / 공백** — 독점·메이저 계약 공백, 라이선스 여지, 접촉 난이도.

> ②·③처럼 맥락이 핵심인 축은 단순 수치로 환원하지 않는다. 종합 코멘트가 최종 판단의 무게중심.
> 위 4축의 **운영 기준**은 바로 아래 「아티스트 픽업 기준」 6항목으로 구체화된다.

## 아티스트 픽업 기준 (정본)
> 원래 출처는 옵시디언 "아티스트픽업" 노트였으나 **이제 이 문서가 정본**이다. 발굴·점수화는 항상 아래 기준을 따른다. (운영 구현 = `artist-pickup` 스킬)

**공통 규칙**
- **전부 소프트 가중 · 하드 탈락선 없음** — 어떤 기준도 후보를 자동 탈락시키지 않는다. (결정 #002)
- **타깃 = 서양권 우선** — 중화권은 이미 중국 시장을 섭렵 → 화이트스페이스는 *서양 힙합/R&B/POP을 중국에 들여오는 것*. (결정 #003)
- **나이는 점수 항목이 아니라 저장·표시 항목** (타겟 오디언스 산정용).
- **정규화**: 노트가 명시 기준을 준 항목(IG 50만↑, 장르 우선)은 **절대 앵커**, 연속 수치(스트리밍·팔로워·카탈로그)는 **풀 내 상대**.

**점수 기준 (4축)**

| # | 기준 | 비중 | 원문 정의 → 읽는 법 |
|---|------|:---:|------|
| 1 | **Spotify + 주요 글로벌 플랫폼 스트리밍** | 30% | 공개 스트리밍 실적. Spotify에 더해 Apple Music·YouTube Music 등 다른 글로벌 플랫폼 수치가 있으면 함께 반영. **정확성 1순위 데이터** — 가장 정확해야 한다. |
| 2 | **장르** | 30% | **힙합·R&B·POP 우선**(가중 높음). 나머지 장르도 배제 아님, 우선순위만 낮음. |
| 3 | **SNS 팔로워** (IG 중심) | 20% | IG 50만↑ = 좋은 등급(절대 앵커). 그 아래는 규모 + 풀 내 상대. IG 없으면 YT 대체. |
| 4 | **카탈로그 규모 + 시장 반응** | 20% | 발매 곡·앨범 수 + "지금까지 어떤 결과물을 냈고 반응이 어땠는지" 분석. |
| — | (나이) | 점수 X | 저장·표시 항목. 타겟 오디언스 산정용. |

> **비중 근거** (결정 #006): 스트리밍·장르 = 30%, 팔로워·카탈로그 = 20%. 기존 6축의 상대 비율(높음 21% : 중간 14%)을 4축으로 재정규화. "음악·스트리밍·세일즈 데이터가 가장 중요하고 정확해야 한다"는 원칙 유지.

**비점수 영역 (의견만, 점수 X)** — 라이선스 판단에 중요하지만 한 줄 숫자로 환원하면 왜곡되는 정성 영역. 각 아티스트마다 한 줄 의견 + 근거를 달아 스코어카드에 "참고 의견(점수 미반영)"으로 표시. (결정 #006)

| 영역 | 의견에 담을 것 |
|------|------|
| **인디 vs 메이저** | 소속과 직접 계약 용이성. 메이저 = 레이블 경유 필요, 인디 = 직접 계약 가능 → 인디가 유리. |
| **국적 · 중국 활동 가능성** | 중국 활동 이력·검열/정치 리스크. 활동 제한되면 프로모션 자체가 불가. 서양권 타깃 맥락에선 "중국에 들여올 수 있나"로 읽는다. |

## 데이터 전략
- **소스**: 공개 스트리밍/소셜 플랫폼(Spotify, YouTube, SoundCloud, TikTok 등) + 외부 차트·데이터 벤더(Chartmetric 등).
- **획득 원칙**: *수단·방법을 가리지 않는다.* 파이썬 API 클라이언트·스크레이퍼·외부 깃헙 스킬·MCP·수동 입력 등 무엇이든, 데이터를 **코어 모델로 정규화**만 하면 어댑터가 된다.
  - 단, 항상 **합법·각 플랫폼 ToS·저작권·개인정보 범위 내**에서. (→ [CLAUDE.md](./CLAUDE.md) 원칙 2)

## 구조 개념 *(개념만 — 아직 코드 없음)*
- **코어 (안정)**: 데이터 모델 + 4축 스코어링. 프로젝트의 두뇌.
- **획득 / 어댑터 (기회주의적)**: 소스별 수집 → 코어 모델로 정규화.
- **딜리버리 스킬 (온디맨드)**: 점수화 데이터 → 노션/엑셀/메일/리포트/대시보드. 맥락 따라 그때그때 스킬로 제작.

- 폴더 골격·코드 스캐폴딩·웹 대시보드·DB → 전부 **필요해질 때** 추가.
- 현재 단계: **CLAUDE.md / CONTEXT.md로 방향만 세우고 점진적으로 키운다.**

## 용어집
- **어댑터(adapter)** — 한 소스에서 데이터를 가져와 코어 모델로 정규화하는 모듈/수단.
- **딜리버리 스킬** — 점수화 결과를 특정 형식으로 내보내는 Claude Code 스킬.
- **4축 점수** — 성장세 · 충성도 · 적합성 · 계약여지 (개념 논지).
- **픽업 기준 (4점수축 + 2의견)** — 점수: 스트리밍 · 장르 · 팔로워 · 카탈로그. 비점수 의견: 인디/메이저 · 국적/중국활동. 4축 논지의 *운영* 기준 (→ 「아티스트 픽업 기준」 섹션). (결정 #006으로 6항목 → 4점수+2의견 개편)
- **코어(core)** — 수집·분석·점수화의 안정적 중심 로직.

### #003 (2026-06-18) — 타깃 논지 & 발굴 데이터 획득법 (첫 발굴 회고)
- **타깃 = 서양권 우선 (화이트스페이스 전략).** 중화권 아티스트는 이미 중국 시장을 섭렵 → 라이선스 가치가 낮음. 반대로 **서양권 힙합/R&B/POP 아티스트를 중국 시장에 들여오는 것**이 공백을 메우는 발굴이다. → 발굴 렌즈 기본값을 "서양권 라이징 인디"로.
