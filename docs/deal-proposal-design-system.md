## 1. 페이지 & 그리드

- **용지**: A4 (210 × 297 mm = 11906 × 16838 DXA) — 중국·글로벌 공용 표준
- **여백**: 상 28mm / 하 25mm / 좌우 24mm (에디토리얼답게 넉넉히). DXA ≈ 상 1588 / 하 1440 / 좌우 1361
- **본문 폭**: 약 162mm (단일 컬럼). 표·콜아웃도 이 폭에 맞춤
- **세로 리듬**: 문단 간격 = 본문 1줄(약 8pt after). 섹션 간격 = 24pt before
- **정렬**: 본문 좌측 정렬(양끝맞춤 금지 — 에디토리얼 가독성 우선)

## 2. 컬러 토큰 (중립 프리미엄)

| 토큰 | HEX | 용도 |
|---|---|---|
| `ink` | **#1A1A1A** | 본문·제목 기본 텍스트 |
| `ink-soft` | **#6B6B6B** | 보조 텍스트·라벨·캡션 |
| `paper` | **#FFFFFF** | 기본 배경 |
| `panel` | **#F7F4EF** | 콜아웃/요약 박스 배경 (따뜻한 오프화이트) |
| `hairline` | **#DAD6CE** | 괘선·표 보더 (얇은 한 줄) |
| `accent` | **#6E2A33** | 절제된 에디토리얼 액센트(딥 버건디) — 섹션 번호·핵심 수치·표지 룰에만 |
| `brand-red` | **#C20C0C** | NetEase 레드 — **푸터 워드마크 1곳만** 옵션. 본문엔 쓰지 않음 |

규칙: 한 페이지에서 `accent`는 **포인트로만**(핵심 1~2곳). 색을 면(面)으로 깔지 않는다. 강조는 색보다 **굵기·크기·여백**으로.

## 3. 타이포그래피 토큰

에디토리얼 원칙 = **세리프(산문) + 산세리프(데이터/라벨)** 페어링. docx 보편 폰트만 사용.

| 역할 | 폰트 | 크기/스타일 | 색 |
|---|---|---|---|
| Cover Title | **Georgia** | 30pt, Regular, 자간 살짝 | `ink` |
| Eyebrow/Label (라벨) | **Arial** | 9pt, 대문자, 자간 +1.5, Bold | `ink-soft` |
| H1 (섹션 제목) | **Georgia** | 18pt, Semibold | `ink` |
| H2 (소제목) | **Arial** | 12pt, Bold | `ink` |
| Body (본문 산문) | **Georgia** | 11pt, 줄간 1.45 | `ink` |
| Data/Table | **Arial** | 10pt, 줄간 1.2 | `ink` |
| Key Figure (핵심 금액) | **Georgia** | 20pt, Semibold | `accent` |
| Caption/Legal | **Arial** | 8pt | `ink-soft` |

> 폰트 fallback: Georgia 없을 시 Cambria → 본문 serif. Arial 없을 시 Helvetica/Calibri.
> 본문은 **세리프(Georgia)**, 숫자·표·라벨은 **산세리프(Arial)** — 이 분리를 끝까지 지킨다.

## 4. 컴포넌트 스펙

### 4.1 표지 (Cover)
- 상단: `STRICTLY PRIVATE & CONFIDENTIAL` (Eyebrow, `ink-soft`)
- 중앙: 제안 제목(Cover Title) + 그 아래 `accent` 색 hairline 1줄(폭 48mm)
- 하단 블록(Label/Data 페어): Prepared For / Prepared By / Date / Subject
- 장식 최소. 여백이 주인공.

### 4.2 섹션 헤더
- `01` 형태의 섹션 번호(Arial, `accent`) + H1 제목(Georgia)
- 제목 아래 본문 폭 전체 `hairline` 1줄
- 바로 아래 **리드 문장**(Body, 1~2줄)으로 "이 섹션이 뭘 말하는지" 먼저

### 4.3 콜아웃 / 요약 박스 ("결론 먼저")
- 배경 `panel`, 좌측 3pt `accent` 세로 바, 내부 여백 12pt
- 용도: 섹션 첫머리 핵심 요약, Option A/B 한 줄 요약, 보장 사항
- 보더 없음(배경+좌측 바로만 구분) — 가벼운 에디토리얼 룩

### 4.4 데이터 표
- 보더: `hairline` 가로선만(세로선 없음 → 에디토리얼). 헤더행 하단만 약간 진하게
- 헤더행: 배경 `panel`, Arial 10pt Bold
- 셀 패딩: 상하 6pt / 좌우 10pt. 행 높이 넉넉히
- 숫자/금액 우측 정렬, 라벨 좌측 정렬

### 4.5 Option A vs B 비교
- 2열 카드형(좌 A / 우 B) 또는 나란한 2열 표
- 각 카드 머리에 **Key Figure**(예: `$250,000`)를 크게
- 하단에 "→ 한 줄 결론"(context §4의 비전문가용 프레이밍 그대로)

### 4.6 보장 체크리스트
- 체크 글리프(✓)는 `accent`, 항목 텍스트 Body
- 불릿 아이콘 직접 입력 금지 → docx numbering(체크 스타일)로

### 4.7 푸터
- 좌: 기밀 문구(Caption). 우: 페이지 번호(Caption)
- 브랜드 워드마크는 **옵션**으로 푸터 한 줄에만(중립 프리미엄이라 절제)

## 5. 언어·보이스 규칙

- **결론 먼저**: 모든 섹션은 요약 콜아웃으로 시작.
- **짧은 문장**: 한 문장 한 메시지. 수동태·법률투 최소화.
- **정확한 용어 유지**: MG, sync, perpetual, recoupable, advance 등은 **정확히 그대로**. 풀어쓰느라 부정확해지지 않는다. (설명은 대면)
- **숫자는 또렷하게**: 금액·퍼센트·기간은 굵게/크게. 통화·단위 항상 표기.
- **과장 금지**: 단정 못 할 건 단정하지 않는다 (context의 explainability 원칙).

## 6. docx 구현 노트 (생성 시)

- docx-js 또는 unpack→XML 편집. 페이지 A4 명시(기본 A4지만 명시).
- 스타일을 **이름으로 정의**(Heading1=Georgia18 등)해서 일관 적용 — 인라인 서식 남발 금지.
- 표는 `WidthType.DXA` + 셀 dual-width. 세로 보더 제거(가로 hairline만).
- 색은 §2 HEX 그대로. 액센트 남용 금지.
- 폰트는 §3 + fallback. 임베딩은 하지 않되 보편 폰트라 안전.
